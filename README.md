## 🎯 스터디 목표

## 🧑‍💼 기본 진행 방식 (모든 회차 공통)

### 역할 배정
- **면접자 1명**: 전체 파이프라인 책임
- **면접관 2명**: 질문으로 면접자를 이끌어 답을 유도

### 면접자 준비물
1. **1페이지 설계 요약** (그림 포함)
2. **전제 조건 3가지** 명시
3. 책 설계에서 **본인이 바꿔본 포인트 1~2개**

### 면접관 역할
| 면접관 | 담당 영역 |
|:---:|:---|
| M1 | 문제 정의 / 데이터 / 모델 중심 질문 |
| M2 | 평가 / 서빙 / 운영 / 스케일 중심 질문 |

### 면접관 준비물
1. **질문 목록**: 면접자를 이끌어낼 질문들
2. **공유 내용**: 공부하면서 이건 공유하고 싶다 싶은 내용

## 📝 진행 순서 (회차당 약 60분)

1. **면접 시작** (5분)
   - 면접자가 주제 소개 및 전제 조건 설명

2. **설계 발표** (15분)
   - 면접자가 자료 없이 파이프라인 설명
   - 문제 정의 → 데이터 → 모델 → 평가 → 서빙

3. **면접관 질의응답** (25분)
   - M1: 문제 정의 / 데이터 / 모델 관련 질문
   - M2: 평가 / 서빙 / 운영 / 스케일 관련 질문
   - 면접관은 답을 주지 않고 질문으로 유도

4. **피드백 및 토론** (10분)
   - 면접관들이 준비한 공유 내용 발표
   - 책 내용과 비교, 개선점 토론

5. **마무리** (5분)
   - 다음 회차 역할 확인
   - 산출물 업로드

## 📚 참고: 7단계 ML 시스템 설계 프레임워크

책에서 제시하는 설계 접근법:

1. **요구사항 명확화** (Clarify Requirements)
2. **ML 문제로 구조화** (Frame as ML Problem)
3. **데이터 준비** (Data Preparation)
4. **모델 개발** (Model Development)
5. **평가** (Evaluation)
6. **서빙** (Serving)
7. **모니터링 및 운영** (Monitoring & Operations)
